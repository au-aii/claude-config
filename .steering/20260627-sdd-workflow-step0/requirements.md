# Requirements — SDD ワークフロー Step0

## 概要

ADR 0001 に基づき、SDD ワークフローの一貫性を阻む2つの穴を塞ぐ。
実装（エージェント統合コードの変更）は含まない。

## ユーザーストーリー

- **As a** ワークフロー利用者
- **I want** 検証段階で「どのエージェントを呼ぶか」が一意に決まる状態
- **So that** 経験や記憶に頼らず、ドキュメントを見れば正しく使える

---

- **As a** ワークフロー利用者
- **I want** spec / .steering の置き場所が一意に定まっている状態
- **So that** 別リポジトリに移動しても同じ手順で作業できる

## 受け入れ条件

### 穴A（検証入口の統合）

- [ ] 実装検証・コードレビュー・ドキュメントレビュー・PR レビューそれぞれの「呼び出し場面」が
      設計書に一意に定義されている
- [ ] 既存の 4 エージェント（implementation-validator / code-reviewer / doc-reviewer /
      reviewer-agent）をいつ使うかが重複なく記述されている
- [ ] agents / commands / skills / settings / CLAUDE.md は**変更しない**

### 穴B（配置の一意化）

- [ ] spec（永続ドキュメント）と .steering（作業単位ドキュメント）の所有権ルールが
      1 箇所に明文化されている
- [ ] リポジトリ間で揺れていた配置ルールが設計書に集約されている

## 制約事項

- このスコープはドキュメント作成のみ（agents / commands / skills / settings 変更禁止）
- 公開リポジトリのため、絶対パスや私用リポジトリ名を含めない
- 既存の `.steering/20260620-closed-loop-agent-sync/` は変更しない
